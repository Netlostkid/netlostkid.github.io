如何配置和使用代理？

代理软件：v2rayN 下载及入门使用教程

v2rayN 是 v2ray 系列客户端专为 Windows 电脑设计的图形化操作应用软件，具备强大功能并支持多种代理协议，包括VMess、VLESS、Trojan、Socks、Shadowsocks等，快速实现代理上网的功能。

下载

v2rayN 官网下载地址：https://github.com/2dust/v2rayN/releases。

因软件源代码托管于 Github ， 下载速度感人(0.1kb/s)😅

安装包选择

在下载安装包栏中有很多 zip 压缩包选择，我们推荐第一次安装的用户下载 v2rayN-With-Core.7z 压缩包。安装时会自动提示安装[Microsoft .NET 6.0 Desktop Runtime]。

  • v2rayN-With-Core.zip :包含所有Core文件，如您第一次请下载此包
  • v2rayN-With-Core-SelfContained.7z:包含所有Core文件；独立包，不需要下载安装[Microsoft .NET 6.0 Desktop Runtime]
  • v2rayN-32.zip:不包含Core文件
  • v2rayN-arm64.zip:不包含Core文件

软件安装

压缩包放到C盘的目录Program Files文件夹中，并且解压

进入文件夹，启动v2rayN

右击鼠标，以管理员权限打开

鼠标“左击”选中空白窗口，Ctrl+V把代理节点复制到V2rayN中

显示”成功导入字样“

测试代理节点是否有效

开启”自动配置系统代理“

路由个人推荐选择**”绕过大陆”**

软件变为红色：可翻墙

软件变为蓝色：不可翻墙

路由策略解释：

  • 绕过大陆：国内站点不走代理，国外站点走代理，既省流量，访问国内站点快

  • 黑名单：只通过黑名单设置的域名规则匹配则进行代理

  • 全局：所有站点都走代理

然后就可以使用Google Chrome 浏览器和Google 老师打招呼🙋了，不想使用魔法，可以直接退出即可

补充

系统代理

  1. 清除系统代理：每次启动/重启服务的时候，强制把windows系统（ie）的代理清除掉 -> 不启用代理，无法科学上网
  2. 自动匹配系统代理：每次启动/重启服务的时候，强制设定windows系统（ie）的代理 -> 启动代理，由v2rayN服务（127.0.0.1:10809）进行处理
  3. 不改变系统代理：每次启动/重启服务的时候，什么都不做 -> 遵循系统设定，但是v2rayN服务不起作用
  4. Pac模式：每次启动/重启服务的时候，强制设定windows系统（ie）的代理使用脚本。根据Pac文件中的内容，Direct走本地网络，PROXY进入Core的路由规则再次分流 -> 启动代理，每个请求都根据脚本设定来判断走直连（Direct）还是v2RayN服务进行代理（Proxy）

路由（V2Ray可以通过内部机制判断不同国家或地区的流量）

  1. 绕过大陆（Whitelist）：如果是cn的直接出站（白名单），其他走代理 -> “绕过大陆”进行代理
  2. 黑名单（Blacklist）：如果是被cn禁止的进行代理（黑名单），其他直接出站 ->对“黑名单”进行代理
  3. 全局（Global）：所有入站数据走代理出站 -> “全局”代理

常规做法
系统代理选“自动匹配系统代理”
路由选“绕过大陆”

知识扫盲

什么是代理节点？

在翻墙技术中，vmess、vless、reality 和 h2 都是常见的代理协议或节点，通常用于突破网络审查和访问受限内容。它们的工作原理和使用场景各有不同。以下是这些代理节点的深入浅出解释：

1. VMess (V2Ray Protocol)

概述：

  • VMess 是 V2Ray 项目中的核心协议，主要用于绕过网络审查，提供加密的、抗封锁的代理连接。它是 V2Ray 服务端和客户端通信的协议。
  • VMess 通常用于 V2Ray 和 XRay 等翻墙工具，支持多种传输协议和加密方式。

特点：

  • 加密：VMess 提供强大的流量加密，能够有效防止流量被中间人攻击。
  • 抗封锁性强：VMess 对抗流量分析和深度包检测（DPI）技术具有较强的能力，能够更好地绕过防火墙的监控。
  • 多种传输方式：VMess 支持通过 HTTP、TLS、WebSocket、mKCP 等多种方式传输，增加了代理的灵活性。

使用场景：

  • 适用于需要高隐匿性和抗封锁能力的场景，尤其是绕过严格的网络审查环境。

2. VLESS (V2Ray Less)

概述：

  • VLESS 是 V2Ray 协议的一个改进版本，去除了 VMess 协议中的一些设计缺陷，优化了性能和效率。
  • VLESS 在 V2Ray 中被用作替代 VMess 的协议，它支持更多的加密方式，并增强了性能。

特点：

  • 无状态加密：与 VMess 相比，VLESS 不需要会话的状态信息，简化了通信流程，提升了性能。
  • 更强的抗封锁性：VLESS 更加轻量，能够绕过更多的防火墙封锁，尤其在高并发的情况下表现更好。
  • 支持 HTTP/2 和 QUIC：VLESS 支持新型的传输协议（如 HTTP/2 和 QUIC），这些协议能够提高传输效率和抗封锁能力。

使用场景：

  • 适用于对性能要求高且需高效绕过封锁的用户，特别是在网络条件较差的地区。

3. Reality

概述：

  • Reality 是一种由 XRay 项目提出的代理协议，它是基于 VMess 和 VLESS 的改进协议，专注于提升隐蔽性、抗封锁性和性能。
  • Reality 协议通过类似 TLS 握手的方式加密和隐藏流量，使得其更难以被深度包分析（DPI）检测出来。

特点：

  • TLS 伪装：Reality 使用 TLS 握手伪装，使得流量像普通的 HTTPS 流量一样，能有效绕过 HTTPS 协议层的封锁。
  • 非常强的抗封锁性：由于其类似 HTTPS 的流量特征，防火墙很难通过 DPI 检测识别 Reality 流量，从而大大提高了绕过封锁的成功率。
  • 低延迟：Reality 通过优化协议结构，减少了延迟，并能保持良好的网络性能。

使用场景：

  • 适用于封锁严格的环境，特别是需要隐藏代理流量特征的高隐蔽性使用场景。

4. H2 (HTTP/2)

概述：

  • H2 是基于 HTTP/2 协议的一种代理方式。HTTP/2 是 HTTP 协议的升级版，主要用于提高 web 页面加载速度，它可以用于绕过某些防火墙封锁。
  • 在代理应用中，H2 可以用于伪装流量，使得其看起来像普通的 HTTPS 流量，特别适合在无法使用传统翻墙协议的情况下使用。

特点：

  • 高效传输：相比 HTTP/1.x，HTTP/2 支持多路复用，可以并行处理多个请求，减少了延迟，提高了吞吐量。
  • 流量伪装：H2 可以通过标准的 HTTPS 加密通信，使得代理流量更难以被检测和封锁。
  • 简化配置：H2 协议的实现相对简单，能够较快速地部署，并适合低延迟的需求。

使用场景：

  • 适用于有 HTTPS 流量封锁的环境，或者对延迟敏感的应用场景。它通过 HTTPS 伪装隐藏流量，能够有效绕过大部分的封锁和监控。
